# 天气卡片网站规范

## 1. 项目概述

- **项目名称**: Weather Card
- **类型**: 单页静态网站
- **功能**: 展示精美天气卡片，支持搜索城市天气
- **目标用户**: 需要嵌入天气组件的开发者/普通用户

## 2. UI/UX 规范

### 布局结构

- 页面居中显示，最大宽度 480px
- 搜索框 + 天气卡片垂直排列
- 响应式：移动端 padding 16px，桌面端 24px

### 视觉设计

**配色方案 (深色渐变背景)**
- 背景: 渐变 `linear-gradient(135deg, #1a1a2e 0%, #16213e 50%, #0f3460 100%)`
- 卡片背景: `rgba(255, 255, 255, 0.1)` + backdrop-blur
- 主文字: #ffffff
- 次要文字: rgba(255, 255, 255, 0.7)
- 强调色: #ffd700 (金色)

**字体**
- 标题: "Noto Sans SC", sans-serif
- 数字: "DM Sans", sans-serif
- 大小: 温度 72px，日期 18px，城市 28px

**间距**
- 卡片圆角: 24px
- 内边距: 32px
- 元素间距: 16px

**动效**
- 卡片入场: fadeInUp 0.6s ease-out
- 温度数字: 计数动画
- 天气图标: 轻微浮动动画

### 组件

1. **搜索框**
   - 圆角输入框，带搜索图标
   - 聚焦时发光效果
   
2. **天气卡片**
   - 城市名称 + 日期
   - 天气图标 + 描述
   - 当前温度 (大字)
   - 最高/最低温度
   - 湿度/风速信息

3. **天气图标**
   - 使用 emoji 或 Lucide 图标
   - 晴 ☀️ / 阴 🌥️ / 雨 🌧️ / 雪 ❄️ / 多云 ⛅

## 3. 功能规范

### 核心功能
- 输入城市名称搜索天气
- 使用 wttr.in API 获取天气数据 (无需 API key)
- 展示完整天气信息

### 用户交互
- 输入城市 → 按回车或点击搜索
- 默认显示深圳天气
- 加载状态显示骨架屏

### 数据处理
- API: `https://wttr.in/{城市}?format=j1`
- 解析 JSON 提取温度、天气描述、湿度、风速

## 4. 验收标准

- [ ] 页面加载显示默认天气 (深圳)
- [ ] 搜索框可输入城市名并回车搜索
- [ ] 天气卡片正确显示温度、湿度、风速
- [ ] 天气图标根据天气状况变化
- [ ] 动效流畅
- [ ] 移动端适配正常

## 5. 技术栈

- 纯 HTML + CSS + JavaScript
- Vercel 部署
